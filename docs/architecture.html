System Architecture

Portfoli-AI follows a modular architecture that separates the AI orchestration logic from the web serving and data persistence layers. This ensures a scalable and maintainable codebase.

High-Level Workflow

The lifecycle of a portfolio generation event involves three major stages: extraction, augmentation, and synthesis.

  • Ingestion: The user uploads a PDF. main.py handles the upload and stores the raw text.
  • Graph Execution: generator.py triggers the LangGraph workflow to parse and expand data.
  • Persistence: The final JSON structure is saved to SQLite, and extracted knowledge is upserted to ChromaDB.

The Data Transformation Lifecycle

Portfoli-AI is essentially a sophisticated data transformation pipeline. The goal is to take high-density, low-engagement data (a PDF) and turn it into high-engagement, interactive content.

1. De-structuring (Ingestion)

Raw PDF binary data is read using PyPDF. This produces a single, large string of unformatted text.

2. Structuring (Extraction)

The LLM acts as an "unstructured-to-structured" converter. The output is a JSON object with well-defined keys, but the content is still essentially a direct mirror of the resume.

3. Synthesis (Content Generation)

This is where the magic happens. The LLM processes the structured JSON and expands it. For example, a project listed as "React App" might be expanded into "Developed a high-performance React-based dashboard with real-time state management."

The Graph State Object

The GraphState is the single source of truth during the transformation. It ensures that each node knows exactly what it needs to do and what the previous node accomplished.

generator.py 
class GraphState(TypedDict):
    resume_text: str       # Input from UI
    structured_data: Dict  # Output from extraction_node
    portfolio_content: Dict # Output from generation_node
    error: str

Future-Proofing: Error Handling

By using a graph, we can easily add "conditional edges" in the future. For example, if the structured_data is missing a name, the graph could loop back to the extractor with a more specific prompt before moving to the generator.
